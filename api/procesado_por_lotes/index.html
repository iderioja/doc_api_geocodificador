PROCESADO POR LOTES


Aunque en la hoja de ruta del proyecto está previsto incluir un método de procesamiento por lotes, no existe por el momento una opción específica para procesar datos a partir de la información de direcciones contenida en un fichero (1).

La operativa provisional para procesar un conjunto de datos relacionados en un fichero, debe consistir por el momento en realizar tantas llamadas a la API como direcciones se quieran geocodificar.

A modo de ejemplo hemos preparado un script de Python que lee los datos de direcciones contenidas en un fichero CSV que utiliza como separador de campos el signo ;(2), interroga secuencialmente a la API, y escribe en otro fichero CSV los datos de salida junto con sus coordenadas.

El procesamiento de datos en este caso requiere tres ficheros:


Un fichero CSV de nombre direcciones-input.csv que contiene los datos de las direcciones a geocodificar, estructurado conforme con los siguientes atributos:

NOMBRE DEL ATRIBUTO CONTENIDO
_count (Número de fila o registro)
name (Texto y número de la dirección)
locality (Población)
localadmin (Municipio)
region La Rioja
macroregion La Rioja
country España

En nuestro ejemplo este es el contenido del fichero direcciones-input.csv

_count;name;locality;localadmin;region;macroregion;country
0;Calle Marqués de Murrieta 76;Logroño;Logroño;La Rioja;La Rioja;España
1;Calle General Vara de Rey 3;Logroño;Logroño;La Rioja;La Rioja;España
2;Glorieta Quintiliano 1; Calahorra;Calahorra;La Rioja;La Rioja;España
3;Carretera de Alcanadre 2;Los Molinos de Ocón;Ocón;La Rioja;La Rioja;España
4;Carretera Miranda Fonzaleche, 11;Villaseca;Fonzaleche;La Rioja;La Rioja;España
5;Plaza de la Paz, 1;Haro;Haro;La Rioja;La Rioja;España


Un fichero denominado geocoder_csv.py que contiene el código python del script:

import csv
import requests
def geocoding(name, locality, localadmin, region, macroregion, country):
    url = "https://geocoder-5-ign.larioja.org/v1/search"
    params = dict(
        text=",".join([name, locality, localadmin,
                      region, macroregion, country]),
        size=1
    )
    resp = requests.get(url=url, params=params)
    data = resp.json()
    if data['features']:
        result = data['features'][0]['geometry']['coordinates']
        print("encontrado: ", params)
    else:
        print("fallo: ", params)
        result = [0, 0]
    return result
with open('direcciones-input.csv') as csv_file:
    csv_reader = csv.reader(csv_file, delimiter=',')
    line_count = 0
    for row in csv_reader:
        if line_count == 0:
            print(f'Nombres de columnas del fichero CSV: {", ".join(row)}')
            with open('direcciones-output.csv', mode='w') as csv_file:
                csv_writer = csv.writer(
                    csv_file, delimiter=';', quotechar='"', quoting=csv.QUOTE_MINIMAL)
                csv_writer.writerow(
                    ['_count', 'name', 'locality', 'localadmin', 'region', 'macroregion', 'country', 'lat', 'lon'])
            line_count += 1
        else:
            print(
                f'\t Procesando dirección... {row[1]}.')
            result = geocoding(row[1], row[2], row[3], row[4], row[5], row[6])
            with open('direcciones-output.csv', mode='a') as csv_file:
                csv_writer = csv.writer(
                    csv_file, delimiter=';', quotechar='"', quoting=csv.QUOTE_MINIMAL)
                csv_writer.writerow(
                    [row[0], row[1], row[2], row[3], row[4], row[5], row[6], result[0], result[1]])
            line_count += 1
    print(f'Processed {line_count} lines.')


Y un tercer fichero adicional denominado requirements.txt que contiene el siguiente texto

requests


Una vez ejecutado el script, este devuelve un fichero CSV denominado direcciones-output.csv que incluye los datos solicitados, conforme a la siguiente estructura:

NOMBRE DEL ATRIBUTO CONTENIDO
_count (Número de fila o registro)
name (Texto y número de la dirección)
locality (Población)
localadmin (Municipio)
region La Rioja
macroregion La Rioja
country España
lat (Latitud de la dirección)
lon (Longitud de la dirección)

En el ejemplo presente el contenido del fichero direcciones-output.csv es el siguiente:

NOTAS:

(1) La API Pelias no dispone actualmente de ningún método diseñado para el procesamiento de datos por lotes.
(2) Se ha optado por utilizar el signo «;» como signo separador de los atributos del fichero CSV ya que algunos textos de direcciones utilizan la coma como separador de los datos de calle y número, lo que podría genera conflictos de intepretación.
